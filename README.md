# CareBook: Web-Based Patient Appointment Booking System

## 📌 Project Overview
**CareBook** is an end-to-end software product design aimed at modernizing clinical scheduling operations. The project transitions **RdxCare Private Clinic (Indore)** from an error-prone manual paper ledger system to a fully automated, synchronized digital portal. This solution eliminates resource vulnerabilities like duplicate appointments, long customer wait times, and a total lack of scheduling visibility for medical staff.

* **Role:** Lead Business Analyst / Product Owner
* **Artifacts Included:** Business Requirement Document (BRD), Agile Backlog (Jira), Use Case Models, Interface Wireframes
* **Target Domain:** Health Tech / Clinical Resource Management

---

## 🛠️ core Core Competencies Demonstrated
* **Requirements Elicitation:** Formulated targeted stakeholder discovery questions to establish deep functional boundaries.
* **Agile Framework Delivery:** Authored 10 robust User Stories utilizing strict **Gherkin Syntax (Given/When/Then)** for testing validation.
* **System Design Modeling:** Structural Use Case specification mappings to isolate user permissions and database system requirements.
* **UI/UX Field Mapping:** Designed high-fidelity component layouts mapping system logic directly to user interfaces.

## 🏃‍♂️ Agile Project Management (Jira Software)
The entire functional scope was broken down, sized using Fibonacci story points, categorized into 4 core functional **Epics**, and managed across an active delivery board.

* **Epic 1:** Patient Authentication & Profiling Management.
* **Epic 2:** CareBook Scheduling & Transaction Management.
* **Epic 3:** Staff Dashboards & Resource Overrides.
* **Epic 4:** Notification Dispatches & Business Intelligence Reporting.

### Active Sprint Pipeline Setup
To mimic real software development life cycles, a 2-week active sprint iteration was initiated to isolate core MVP dependencies.

* **CB-1 (Done):** New Patient Account Registration & Verification.
* **CB-2 (In Progress):** Dynamic Doctor Availability Slot Roster View.
* **CB-3 (To Do):** Patient Appointment Self-Booking Transaction.
